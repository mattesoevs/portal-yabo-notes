# portal-yabo-notes

一个用于归档和发布多个独立 HTML 页面的仓库。这些页面内容相互独立，不针对任何特定域名或网站，仅作为静态资源存储与展示。

## 项目简介

本项目旨在整理和存放若干独立编写的 HTML 文件，方便通过 GitHub Pages 或其他静态托管服务进行浏览与分享。仓库中的每个页面均为自包含的 HTML 文档，不含外部依赖或特定平台绑定。

## 目录结构

```
portal-yabo-notes/
├── pages/          # 存放所有 HTML 页面文件
├── assets/         # 页面所需的静态资源（如图片、样式、脚本）
└── README.md       # 本文件
```

- `pages/`：每个 HTML 文件对应一个独立页面，文件名即页面标识。
- `assets/`：按需存放页面引用的资源文件，建议按页面或类型组织。

## 页面归档说明

- 所有页面均为纯前端实现，不涉及后端服务或数据库。
- 页面内容可能包含笔记、工具、演示或实验性项目，具体用途参见各页面头部注释。
- 归档内容不定期更新，不保证所有页面持续可用或内容完整。

## 使用方式

1. 克隆本仓库到本地：
   ```
   git clone https://github.com/your-username/portal-yabo-notes.git
   ```
2. 直接在浏览器中打开 `pages/` 目录下的任意 HTML 文件即可查看。
3. 若启用 GitHub Pages，可将 `pages/` 目录设置为站点根目录，或使用 `docs/` 等自定义路径。

## 维护说明

- 欢迎提交 Issue 或 Pull Request 修正页面错误、补充资源或提出改进建议。
- 新增页面请统一放入 `pages/` 目录，并尽量保持文件自包含。
- 不建议在页面中添加外部跟踪、广告或与内容无关的第三方资源。
